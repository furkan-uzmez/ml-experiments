# Phase 4 Plan: nnU-Net Integration

## Goal
Integrate `nnUNetv2` into the benchmark. This requires mapping our ISIC 2018 dataset format into nnU-Net's strict raw data format, overriding its internal cross-validation mapping to enforce our `primary_split.json`, and encapsulating its training/inference via wrapper scripts.

## Requirements Covered
NNUNET-01, NNUNET-02, NNUNET-03, NNUNET-04, NNUNET-05, NNUNET-06

## Architecture & Integration

```
artifacts/nnunet/
├── nnUNet_raw/
│   └── Dataset500_ISIC2018/
│       ├── imagesTr/      # 3 channels independent (e.g., case_000_0000.png)
│       ├── labelsTr/      # Binary masks
│       └── dataset.json
├── nnUNet_preprocessed/
│   └── Dataset500_ISIC2018/
│       └── splits_final.json  # FORCES our custom train/val split
└── nnUNet_results/
```

## Tasks

### 1. Format Conversion (`src/dataio/convert_to_nnunet.py`)
- Define Dataset ID `500` for ISIC 2018.
- Read `configs/dataset.yaml` and our `split_loader.py`.
- **Modality Extraction**: Since nnU-Net v2 expects each channel as a separate file, we will read the ISIC RGB `.jpg` files and save 3 separate `.png` files (R, G, B) mapping to modalities `_0000`, `_0001`, `_0002` respectively.
- Create `dataset.json` mapping 3 modalities to the single foreground label.

### 2. Custom Split Enforcement (`src/dataio/enforce_nnunet_split.py`)
- nnU-Net by default performs random 5-fold cross-validation.
- We must synthesize `splits_final.json` using our exact `primary_split.json`.
- Place this JSON in the `nnUNet_preprocessed/Dataset500_ISIC2018/` directory *after* running `nnUNetv2_plan_and_preprocess` so it ignores its own split logic.

### 3. Execution Pipeline (`scripts/run_nnunet_benchmark.sh`)
- Set environment variables (`nnUNet_raw`, `nnUNet_preprocessed`, `nnUNet_results`).
- Calls:
  1. `src/dataio/convert_to_nnunet.py`
  2. `nnUNetv2_plan_and_preprocess -d 500 --verify_dataset_integrity`
  3. `src/dataio/enforce_nnunet_split.py`
  4. For seed in [11, 22, 33]:
     - `nnUNetv2_train 500 2d all` (Passing env var seeds)
     - `nnUNetv2_predict -i <test_images_folder> -o <predictions_folder>`
  5. Post-prediction wrapper `src/infer/eval_nnunet.py` to calculate identical metrics using Phase 2 `metrics.py` and `resample.py`.

## Verification & User Feedback
> [!IMPORTANT]
> - By default, nnU-Net uses stochastic augmentations. It has an internal seed variable we can force via python environment arguments `np.random.seed(seed)` in train files, but it's famously stubborn about strictly enforcing deterministic hardware seeds across cudnn. We will attempt strict determinism via OS env variables (`PYTHONHASHSEED`, etc.) but expect slight hardware variance if run on different GPUs. Is this acceptable for the benchmark?
> - Does splitting the RGB JPEG into three distinct PNG format channels (`_0000`, `_0001`, `_0002`) align with your expectation of handling color datasets in nnU-Net? 

Please verify these open questions before proceeding to the code implementations.
