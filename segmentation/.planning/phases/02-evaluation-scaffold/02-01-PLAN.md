# Phase 2 Plan: Evaluation Scaffold

## Goal
Build the shared data loading and evaluation pipeline used by all three model pipelines. By building this first, we ensure all models are evaluated exactly the same way.

## Requirements Covered
EVAL-01, EVAL-02, EVAL-03, EVAL-04, EVAL-05, EVAL-06, EVAL-07, EVAL-08, DATA-04, DATA-05, REPRO-02

## Architecture

```
src/
├── dataio/
│   ├── dataset_index.py  # Maps patient IDs to image/mask paths
│   └── split_loader.py   # Loads primary_split.json and provides train/val/test IDs
├── evaluation/
│   ├── metrics.py        # Computes Dice, IoU, HD95 (medpy)
│   ├── resample.py       # Resizes predictions to match ground truth
│   └── runtime.py        # Decorators/utils for timing and GPU memory tracking
```

## Tasks

### 1. Data I/O Layer
- **`src/dataio/dataset_index.py`**:
  - Read `configs/dataset.yaml`
  - Scan `root_path/images_dir` and `root_path/masks_dir`
  - Map `ISIC_XXXXXXX` stem to absolute paths
  - Provide a function `get_case(patient_id)` returning `{image_path, mask_path}`
- **`src/dataio/split_loader.py`**:
  - Read `splits/primary_split.json`
  - Provide `get_train_ids()`, `get_val_ids()`, `get_test_ids()`

### 2. Evaluation Metrics (`src/evaluation/metrics.py`)
- Implement `compute_metrics(pred_mask: np.ndarray, gt_mask: np.ndarray) -> dict`
- Use `medpy.metric.binary.dc` for Dice, `medpy.metric.binary.hd95` for HD95
- Implement edge case handling specified in `configs/metrics.yaml`:
  - Empty pred & empty GT: Dice = 1.0, HD95 = NaN
  - Empty pred & non-empty GT: Dice = 0.0, HD95 = max_distance or NaN
  - Non-empty pred & empty GT: Dice = 0.0, HD95 = max_distance or NaN

### 3. Resampling Utilities (`src/evaluation/resample.py`)
- Implement `resample_to_reference(pred_mask: np.ndarray, ref_mask: np.ndarray) -> np.ndarray`
- Use SimpleITK or OpenCV for nearest-neighbor resizing back to original ground truth resolution

### 4. Runtime Tracking (`src/evaluation/runtime.py`)
- Implement `@track_inference_time` decorator
- Implement `@track_peak_gpu_memory` decorator (using `torch.cuda.max_memory_allocated`)

## Verification
- Write `tests/test_metrics.py` generating synthetic boxes (perfect match, no match, empty) and `pytest` it to prove edge cases work.
- Write `tests/test_dataio.py` to assert the loader matches the split file counts exactly.
