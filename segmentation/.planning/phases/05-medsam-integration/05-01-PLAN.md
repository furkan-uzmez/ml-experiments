# Phase 5 Plan: MedSAM Integration

## Goal
Integrate the MedSAM foundation model for prompt-based medical image segmentation. This phase evaluates the capability of an interactive/promptable foundation model against fully supervised baselines by feeding it deterministic bounding boxes derived from the ground truth masks.

## Requirements Covered
MEDSAM-01, MEDSAM-02, MEDSAM-03, MEDSAM-04, MEDSAM-05

## Architecture & Integration

```
src/
├── models/
│   └── medsam_model.py          # Wrapper for segment-anything + MedSAM weights
├── infer/
│   ├── prompt_generator.py      # Computes deterministic BBox from GT (w/ padding)
│   └── infer_medsam.py          # E2E Inference: Image + BBox -> Preprocess -> Predict
configs/
└── medsam.yaml                  # Prompt padding, thresholds, model paths
scripts/
└── run_medsam_benchmark.sh      # Bash script to run evaluation natively
```

## Tasks

### 1. Configuration (`configs/medsam.yaml`)
- Define MedSAM ViT-B architecture parameters.
- Define Bounding Box extraction rules (e.g., `padding: 5` pixels to mimic a loose human click).
- Paths for the downloaded MedSAM checkpoint.

### 2. Model Adapter (`src/models/medsam_model.py`)
- Programmatic instantiator for `segment_anything.sam_model_registry`.
- Loads the MedSAM `medsam_vit_b.pth` checkpoint safely.
- Exposes a unified inference method accommodating the Segment Anything input structures (256x256 resizing logic with padding specific to SAM).

### 3. Deteministic Prompter (`src/infer/prompt_generator.py`)
- Reads Ground Truth references from `dataset_index.py`.
- Extracts `[x_min, y_min, x_max, y_max]` using NumPy.
- Expands the box deterministically by the padding specified in `medsam.yaml` without violating image boundaries.

### 4. Zero-Shot Evaluator (`src/infer/infer_medsam.py`)
- Bypasses traditional training. We are evaluating MedSAM strictly purely as a prompt-conditioned zero-shot tool per the benchmark rules.
- Loads `configs/seeds.yaml` (Though MedSAM inference is deterministic, keeping it standard).
- Implements `@track_inference_time` and `@track_peak_gpu_memory`.
- Yields standard `compute_metrics` comparison and saves final `.png` masks to `artifacts/medsam/predictions`.

### 5. Automated Launcher (`scripts/run_medsam_benchmark.sh`)
- Wraps the `infer_medsam.py` evaluation loop.
- Optionally downloads the MedSAM checkpoint if missing.

## Verification & User Feedback
> [!IMPORTANT]
> - MedSAM provides extremely fast evaluation compared to standard training cycles because no backpropagation is required. However, the exact checkpoint (`medsam_vit_b.pth`) is quite large (approx. 350 MB). My `run_medsam_benchmark.sh` will `wget` the weights automatically if it doesn't find them under `artifacts/medsam/weights/`. Is this acceptable?
> - The benchmark protocol mandates that MedSAM will receive exactly 1 loosely padded bounding box per test case. If a test case mask has *multiple* disconnected lesions, standard SAM might only predict one. Our prompt generator will draw a macro-box encompassing *all* components. Shall I implement it this way?
